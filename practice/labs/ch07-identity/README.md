# Chapter 07: Identity - Overview

Understand how identities and permissions are managed in Kubernetes using ServiceAccounts, [RBAC](../../../GLOSSARY.md#rbac), and Admission Controllers.

| Lab Name | Description |
| --- | --- |
| [Lab 01: ServiceAccount Identity](./lab01-serviceaccount-identity/README.md) | Learn how Pods use ServiceAccounts to identify themselves. |
| [Lab 02: RBAC Identity](./lab02-rbac-identity/README.md) | Implement [Role](../../../GLOSSARY.md#role)-Based Access Control to restrict resource access. |
| [Lab 03: Admission Control](./lab03-admission-control-entry-permit/README.md) | Use admission controllers to enforce policies on resource creation. |
| [Lab 04: Admission Configuration](./lab04-admission-control-configuration/README.md) | Configure and manage admission controller plugins in the [API server](../../../GLOSSARY.md#api-server). |
| [Lab 05: ServiceAccount Token Decode](./lab05-serviceaccount-token-decode/README.md) | Retrieve and decode the base64 token associated with a [ServiceAccount](../../../GLOSSARY.md#serviceaccount). |
| [Lab 06: Namespaced RBAC](./lab06-namespaced-rbac/README.md) | Learn how to restrict [RBAC](../../../GLOSSARY.md#rbac) permissions to a specific [namespace](../../../GLOSSARY.md#namespace). |
